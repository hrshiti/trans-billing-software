FUNCTIONAL REQUIREMENT DOCUMENT (FRD)
Project Name: Billing Software
1. Project Overview
The Billing Software is an application designed for Transporters and Garage Owners to manage their daily operations, billing, and customer records efficiently.
The system provides:
•
Role-based login for users (Transport / Garage)
•
Billing & invoice generation
•
Party and transaction management
•
Admin control panel for system management
2. Objectives
•
Simplify billing → Fast invoice creation
•
Manage customers → Party accounts
•
Track operations → Trips / Services
•
Generate invoices → PDF + share
3. User Roles
3.1 Admin
System owner → Full access to manage platform
3.2 User (Role-Based Only)
Operational user → Access based on selected role
User Roles:
•
Transport User → Manages trips & transport billing
•
Garage User → Manages vehicle services & billing
4. Authentication System
4.1 Login
•
Mobile Number → Primary login
•
OTP Verification → Secure access
4.2 Role Selection (User Only)
User selects role at first login:

Transport
•

System loads modules based on selected role
